# Core 项目大纲 (Core Project Outline)

## 1. 项目定位 (Project Positioning)

`core` 模块是 `newbee` 项目的后端核心，旨在为上层业务和未来所有微服务提供稳定、高效、安全的底层能力支持。它采用 API 网关 + RPC 服务的架构模式，实现了核心业务逻辑与对外接口的解耦。

### 1.1. 核心组件

- **`core/api`**: API 网关服务。负责处理外部 HTTP 请求、参数校验、身份认证、权限校验，并将业务逻辑转发给内部 RPC 服务。它是系统对外的唯一入口。
- **`core/rpc`**: 核心 RPC 服务。负责实现核心业务逻辑，如用户账户、权限、部门、字典等。它通过 gRPC 与 API 网关通信，并直接与数据库交互。

### 1.2. 技术栈

- **开发框架**: `go-zero`
- **ORM**: `ent`
- **API 定义**: `go-zero API` 语法
- **RPC 定义**: `Protobuf`

## 2. 架构演进方向 (Architectural Evolution)

为了支持业务向 SaaS (软件即服务) 模式转型，并为未来的多业务线扩展提供基础，`core` 模块即将进行一次重要的架构升级：**引入多租户能力**。

### 2.1. 目标

- **数据隔离**: 实现不同租户之间的数据严格隔离，确保数据安全。
- **可扩展性**: 设计一套可复用的多租户方案，让未来新建的微服务能够低成本、快速地支持多租户。
- **统一管理**: 在平台层面提供对租户的统一管理能力。

### 2.2. 实施方案

我们将采用业界主流的 **“共享数据库、共享 Schema、租户ID隔离”** 的方案。通过在数据访问层（ORM）利用 `ent` 的 `Hook` 和 `Mixin` 机制，实现对业务逻辑代码透明的、自动化的数据过滤和填充，从而在底层保障多租户数据安全。

详细的开发任务和迁移计划，请参见 `task_plannings/multitenancy/` 目录下的相关文档。
